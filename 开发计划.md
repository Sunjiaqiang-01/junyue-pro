## 🎯 项目概述

### 业务模型调整说明

**v1.0 → v2.0 重大变更**：

- ❌ 删除：用户注册/登录、订单系统、支付系统、积分系统、评价系统、代理端
- ✅ 保留：技师注册/资料管理、用户端浏览技师（无需登录）、管理端审核管理
- ✅ 新增：客服联系功能、客服配置管理

**核心业务流程**：

```
用户 → 浏览技师信息 → 点击"联系客服预约" → 添加客服微信
→ 客服查看技师联系方式 → 撮合预约 → 平台收取服务费
```

### 项目规模

- **前端页面**: ✅ 15+个页面（用户端5 + 技师端6 + 管理端4+）
- **后端API**: ✅ 62个接口（实际开发超出预期）
- **数据表**: ✅ 20张表（实际开发超出预期11张）
- **核心功能**: ✅ 用户端浏览 + 技师端注册管理 + 管理端审核
- **UI组件**: ✅ 38个定制组件（基于21st.dev）

### 开发优先级

- **P0** (核心功能，必须有): 用户浏览技师 + 客服联系 + 技师注册审核
- **P1** (重要功能，应该有): 技师资料管理 + 管理端数据统计
- **P2** (可选功能，最好有): 技师时间表 + 高级筛选 + 数据导出
